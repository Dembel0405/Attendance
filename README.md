# Система учёта посещаемости студентов

Веб-приложение для автоматизации учёта посещаемости студентов с элементами искусственного интеллекта. Система позволяет импортировать данные из Excel, вести журнал посещаемости и анализировать академические риски.

## Технологии

| Часть       | Стек                                        |
|-------------|---------------------------------------------|
| Frontend    | React 18, TypeScript, Vite, Tailwind CSS    |
| Backend     | Node.js, Express.js                         |
| База данных | SQLite (better-sqlite3)                     |
| Графики     | Recharts                                    |
| Импорт      | xlsx (Excel .xlsx / .xls)                   |
| Авторизация | JWT (jsonwebtoken, bcryptjs)                |

## Установка

### 1. Клонировать репозиторий

```bash
git clone <url-репозитория>
cd Attendance
```

### 2. Установить все зависимости

```bash
# Установить зависимости корневого проекта
npm install

# Установить зависимости сервера
npm install --prefix server

# Установить зависимости клиента
npm install --prefix client
```

Или одной командой:
```bash
npm install && npm run install:all
```

## Запуск

### Режим разработки (сервер + клиент одновременно)

```bash
npm run dev
```

После запуска откроется:
- **Интерфейс:** http://localhost:5173
- **API сервер:** http://localhost:3001

### Запуск по отдельности

```bash
# Только сервер
npm run start:server

# Только клиент (в отдельном терминале)
cd client && npm run dev
```

## Руководство пользователя

### Дашборд

Главная страница показывает общую сводку:
- Количество студентов, студентов в зоне высокого риска, средняя посещаемость
- Столбчатая диаграмма средней посещаемости по группам
- Круговая диаграмма распределения уровней риска
- Список студентов, требующих внимания
- Последние записи посещаемости

### Группы

Управление учебными группами:
1. Введите название группы в поле и нажмите **Добавить**
2. Группа появится в таблице со счётчиком студентов
3. Для удаления группы нажмите **Удалить** — студенты сохранятся, но потеряют привязку к группе

### Студенты

Управление списком студентов:
1. Введите ФИО, выберите группу (необязательно), нажмите **Добавить**
2. Используйте фильтр по группе для просмотра нужного списка
3. Удалить студента можно кнопкой **Удалить** — все его записи посещаемости также удалятся

### Посещаемость

Журнал посещаемости с цветовой индикацией:
- Зелёный — **Присутствовал**
- Красный — **Отсутствовал**

Для ручного добавления записи:
1. Нажмите **Добавить запись**
2. Выберите студента, дату и статус
3. Нажмите **Сохранить**

Для фильтрации используйте панель фильтров: по группе, студенту, диапазону дат.

### Импорт из Excel

Самый быстрый способ наполнить систему данными:

1. Откройте страницу **Импорт**
2. Перетащите файл `.xlsx` в зону загрузки или нажмите для выбора
3. Нажмите **Начать импорт**
4. После импорта отобразится сводка: сколько групп/студентов/записей создано

**Формат Excel-файла:**

| Столбец          | Альтернативные названия     | Допустимые значения             |
|------------------|-----------------------------|----------------------------------|
| `name` (обяз.)   | Имя, ФИО, Студент           | ФИО студента                    |
| `group`          | Группа, Group_name          | Название группы                  |
| `status` (обяз.) | Статус, Присутствие         | present / absent / да / нет / + / - |
| `date` (обяз.)   | Дата, Date_of_attendance    | 2024-09-01 или 01.09.2024       |

Пример строки в Excel:

| name                     | group | date       | status  |
|--------------------------|-------|------------|---------|
| Иванов Иван Иванович     | ИС-21 | 2024-09-02 | present |
| Петров Пётр Петрович     | ИС-21 | 2024-09-02 | absent  |

> Готовый тестовый файл `sample_attendance.xlsx` находится в корне проекта.

### Аналитика (AI модуль)

Система автоматически анализирует каждого студента и определяет:

| Показатель              | Описание                                                          |
|-------------------------|-------------------------------------------------------------------|
| **% посещаемости**      | Доля присутствий от общего числа занятий                         |
| **Уровень риска**       | Низкий (≥80%) / Средний (50–79%) / Высокий (<50%)               |
| **Пропуски подряд**     | Максимальное число последовательных пропусков                    |
| **Тренд**               | Улучшение / Стабильно / Ухудшение (сравнение первой и второй половины записей) |
| **Рекомендация**        | Автоматически сформированный текст на основе всех факторов       |

На странице аналитики можно:
- Фильтровать по группе
- Искать студентов по имени
- Сортировать по уровню риска, посещаемости или имени
- Просматривать круговую и столбчатую диаграммы

## REST API

Базовый URL: `http://localhost:3001/api`

| Метод  | Эндпоинт              | Описание                        |
|--------|-----------------------|---------------------------------|
| POST   | /auth/login           | Вход                            |
| POST   | /auth/register        | Регистрация                     |
| GET    | /groups               | Список групп                    |
| POST   | /groups               | Создать группу                  |
| DELETE | /groups/:id           | Удалить группу                  |
| GET    | /students             | Список студентов                |
| POST   | /students             | Добавить студента               |
| DELETE | /students/:id         | Удалить студента                |
| GET    | /attendance           | Журнал посещаемости             |
| POST   | /attendance           | Добавить запись                 |
| POST   | /import               | Импорт из Excel (multipart)     |
| GET    | /analytics            | AI анализ по всем студентам     |
| GET    | /analytics/:id        | AI анализ конкретного студента  |

Параметры фильтрации для GET /attendance и GET /analytics:
- `group_id` — фильтр по группе
- `student_id` — фильтр по студенту
- `date_from`, `date_to` — диапазон дат (формат YYYY-MM-DD)

## Возможные проблемы

### Ошибка при установке better-sqlite3

Если возникает ошибка компиляции на шаге `npm install --prefix server`, убедитесь, что используется Node.js версии 18+ и что установлены Python и Visual Studio Build Tools (Windows).

Для Windows можно установить инструменты сборки:
```bash
npm install -g windows-build-tools
```

### Порт уже занят

Если порт 3001 или 5173 уже используется, измените его:

- Сервер: установите переменную окружения `PORT=3002` перед запуском
- Клиент: измените `port` в файле `client/vite.config.ts`
